# Contributing to SizeSpy

First off, thank you for considering contributing to SizeSpy! It's people like you that make this tool great.

## Code of Conduct

By participating in this project, you are expected to uphold a welcoming and respectful environment. Please be professional and courteous in all interactions.

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the open issues to see if the problem has already been reported. When reporting a bug, include:
* Your Windows version and OS build.
* Steps to reproduce the behavior.
* What you expected to happen vs what actually happened.

### Suggesting Enhancements

Enhancement suggestions are highly welcomed! Please detail:
* The goal of the enhancement.
* Why it would be useful to most users.
* A clear description of the proposed feature.

### Pull Requests

We welcome bug fixes, feature ideas, and test enhancements!

1. Fork the repo and create your branch from `main`.
2. Make your changes and test them locally.
3. Ensure your batch code is readable and adheres to standard practices.
4. Issue that pull request! Please include a clear description of the changes you made.

## Performance Optimization

If you are suggesting a performance optimization (we love those!):
* Focus on measurable improvements.
* Keep changes minimal (< 50 lines if possible) to maintain code clarity.
* Add clear comments explaining *why* the optimization works (e.g. bypassing `call set` overhead).
